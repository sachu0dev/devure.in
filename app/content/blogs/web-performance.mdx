## Introduction

Web performance is crucial for user experience and SEO. Google's Core Web Vitals have made performance optimization more important than ever. This guide covers essential techniques to improve your website's performance.

### Core Web Vitals

Google's Core Web Vitals measure three key aspects of user experience:

#### 1. Largest Contentful Paint (LCP)

Measures loading performance. Should be under 2.5 seconds.

```javascript
// Monitor LCP
new PerformanceObserver((entryList) => {
  for (const entry of entryList.getEntries()) {
    console.log("LCP:", entry.startTime);
  }
}).observe({ entryTypes: ["largest-contentful-paint"] });
```

#### 2. First Input Delay (FID)

Measures interactivity. Should be under 100 milliseconds.

```javascript
// Monitor FID
new PerformanceObserver((entryList) => {
  for (const entry of entryList.getEntries()) {
    console.log("FID:", entry.processingStart - entry.startTime);
  }
}).observe({ entryTypes: ["first-input"] });
```

#### 3. Cumulative Layout Shift (CLS)

Measures visual stability. Should be under 0.1.

```javascript
// Monitor CLS
new PerformanceObserver((entryList) => {
  let cls = 0;
  for (const entry of entryList.getEntries()) {
    cls += entry.value;
  }
  console.log("CLS:", cls);
}).observe({ entryTypes: ["layout-shift"] });
```

### Image Optimization

Images are often the largest assets on a page:

#### Next.js Image Component

```jsx
import Image from "next/image";

function OptimizedImage() {
  return (
    <Image
      src="/hero-image.jpg"
      alt="Hero image"
      width={800}
      height={600}
      priority // For above-the-fold images
      placeholder="blur"
      blurDataURL="data:image/jpeg;base64,..."
    />
  );
}
```

#### WebP Format

```html
<picture>
  <source srcset="image.webp" type="image/webp" />
  <img src="image.jpg" alt="Description" />
</picture>
```

### Code Splitting

Split your JavaScript bundles to load only what's needed:

```javascript
// Dynamic imports
const LazyComponent = dynamic(() => import("./LazyComponent"), {
  loading: () => <p>Loading...</p>,
  ssr: false,
});

// Route-based splitting
const HomePage = lazy(() => import("./pages/Home"));
const AboutPage = lazy(() => import("./pages/About"));
```

### Caching Strategies

Implement effective caching:

```javascript
// Service Worker caching
const CACHE_NAME = "my-site-cache-v1";
const urlsToCache = ["/", "/styles/main.css", "/scripts/main.js"];

self.addEventListener("install", (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => cache.addAll(urlsToCache))
  );
});
```

### Database Optimization

Optimize your database queries:

```sql
-- Use indexes
CREATE INDEX idx_user_email ON users(email);

-- Limit results
SELECT * FROM posts LIMIT 10;

-- Use specific columns
SELECT id, title, excerpt FROM posts;
```

### CDN Implementation

Use a Content Delivery Network:

```javascript
// Next.js configuration
const nextConfig = {
  images: {
    domains: ["cdn.example.com"],
  },
  async headers() {
    return [
      {
        source: "/(.*)",
        headers: [
          {
            key: "Cache-Control",
            value: "public, max-age=31536000, immutable",
          },
        ],
      },
    ];
  },
};
```

### Conclusion

Performance optimization is an ongoing process. Monitor your Core Web Vitals regularly and implement these techniques to provide the best user experience possible.
