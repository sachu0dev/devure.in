## Introduction

Linux system administration is a crucial skill for developers and DevOps engineers. Understanding the fundamentals of Linux administration can help you deploy, manage, and troubleshoot applications more effectively.

### Essential Commands

Here are some fundamental Linux commands every developer should know:

#### File System Navigation

```bash
# List files and directories
ls -la

# Change directory
cd /path/to/directory

# Print working directory
pwd

# Create directory
mkdir new_directory
```

#### File Operations

```bash
# Copy files
cp source_file destination_file

# Move/rename files
mv old_name new_name

# Remove files
rm filename

# Remove directory recursively
rm -rf directory_name
```

### Process Management

Understanding how to manage processes is essential:

```bash
# View running processes
ps aux

# Kill a process
kill -9 process_id

# Find process by name
pgrep process_name

# Monitor system resources
htop
```

### System Information

```bash
# Check disk usage
df -h

# Check memory usage
free -h

# Check system load
uptime

# View system information
uname -a
```

### Network Commands

```bash
# Check network connectivity
ping google.com

# View network interfaces
ip addr show

# Check listening ports
netstat -tulpn

# Test port connectivity
telnet hostname port
```

### Package Management

Different Linux distributions use different package managers:

#### Ubuntu/Debian (apt)

```bash
# Update package list
sudo apt update

# Install package
sudo apt install package_name

# Remove package
sudo apt remove package_name
```

#### CentOS/RHEL (yum/dnf)

```bash
# Update system
sudo yum update

# Install package
sudo yum install package_name

# Remove package
sudo yum remove package_name
```

### Conclusion

These are just the basics of Linux system administration. As you work more with Linux systems, you'll discover many more powerful commands and techniques. The key is to practice regularly and understand the underlying concepts.
